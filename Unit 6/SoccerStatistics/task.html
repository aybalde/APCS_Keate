Soccer Statistics

Soccer, a sport popular around the world, can be examined in many different ways. Different skills are required for different players based on their positions. Examining some of the core statistics of these players can help get a better understanding of their ranking among other players of their positions. For a more in depth explanation, please read this article released by Bleacher Report.

Assignment:
You and your group will be assigned one of the positions to code. Only turn in one file per group. We will put all of them together to make our program run collectively. You will be graded on whether your code compiles and runs as expected. You will need to test this on your own in the tester class.

1. Fill in the constructor without parameters to read in the necessary information for your position from the user.

All positions will need to read in a name and the number of games the player has played from the user. This will be used to set the size of the arrays in all of the classes. You will also need to read the values of each array in one by one. For example, in the Striker class, for the bigChanceAttempt array, you will need to read in integers that represent how many big chance attempts were made in each game. Each element of that array represents the number of big chance attempts made in one game.
For the Striker class, you will also need to read the values of each array in one by one. For the bigChanceAttempt array, you will need to read in integers that represent how many big chance attempts the striker made in each game. Each element of that array represents the number of big chance attempts the striker made in one game. For the bigChanceMade array, you will need to read in integers that represent how many big chances the striker completed in each game. Each element of that array represents the number of big chances the striker completed in one game. For the goalsAttempted array, you will need to read in integers that represent how many goals the striker attempted in each game. Each element of that array represents the number of goals the striker attempted in one game. For the goalsMade array, you will need to read in integers that represent how many goals the striker made in each game. Each element of that array represents the number of goals the striker made in one game.
For the GoalKeeper class, you will also need to read the values of each array in one by one. For the goalsAgainst array, you will need to read in integers that represent how many goals were made against the goalkeeper in each game (how many times someone scored against them). Each element of that array represents the number of goals the goalkeeper let through in one game. For the saves array, you will need to read in integers that represent how many saves the goalkeeper made in each game. Each element of that array represents the number of saves the goalkeeper made in one game.
For the Midfielder class, you will also need to read the values of each array in one by one. For the interceptions array, you will need to read in integers that represent how many interceptions the midfielder made in each game. Each element of that array represents the number of interceptions the midfielder made in one game. For the ballRecoveries array, you will need to read in integers that represent how many ball recoveries the midfielder made in each game. Each element of that array represents the number of ball recoveries the midfielder completed in one game.
For the Defenders class, you will also need to read the values of each array in one by one. For the interceptions array, you will need to read in integers that represent how many interceptions the defender made in each game. Each element of that array represents the number of interceptions the defender made in one game. For the ballRecoveries array, you will need to read in integers that represent how many ball recoveries the defender made in each game. Each element of that array represents the number of ball recoveries the defender completed in one game.

2. Create a getName method to return the player's name.

3. Fill in the specified methods to calculate statistics for your position. These differ per position.

Strikers need goalPercentage and bigChancePercentage methods. These should return values between 0 and 1.
Defenders and Midfielders need averageInterceptions and averageBallRecoveries methods. These should return values above 0.
GoalKeepers need a savePercentage method. These should return values between 0 and 1.

4. As a group, fill in the SoccerTeam no parameter constructor. Soccer teams consist of a name, 1 goalkeeper, 3 defenders, 4 midfielders, and 3 strikers. Defenders, Midfielders, and Strikers are held in separate arrays by position. Read in the name of the team, the number of games played, and the number of games won from the user. You may call the no parameter constructor of each of those classes to initialize the GoalKeeper and fill and initialize the players in the Defenders, Midfielders, and Strikers arrays.

5. Make sure your names are commented out near the top of your source code.
